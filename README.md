# 고속병렬파일시스템 멀티모달 데이터 통합 관리 플랫폼 기술 개발

## 개요
본 프로젝트는 외부 **고속병렬파일시스템(HPFS)** 에 저장된 대규모 멀티모달 데이터(Text, Image, Audio, Video)를 **일관된 표준으로 수집·추출·정규화·저장·검색** 하기 위한 통합 플랫폼을 구축하는 기술 개발 과제입니다.

고속 병렬 파일 이벤트를 기반으로 자동 메타데이터 추출 파이프라인을 구성하고, 카탈로그 및 검색 시스템을 통해 사용자에게 고성능 검색 기능을 제공합니다.

## 📋 프로젝트 개요
이 플랫폼은 고속병렬파일시스템에서 발생하는 파일 생성·수정·삭제 이벤트를 실시간 감지하여 **메타데이터 추출 → 카탈로그 저장 → 인덱싱 → 검색 API 제공**이라는 데이터 처리 전주기를 자동화합니다.

주요 구성 요소는 다음과 같습니다:
- 파일 인식/연계 모듈
- 메타데이터 관리 모듈
- 인덱싱/검색 모듈
- API 및 UI 서비스
- 운영·모니터링 모듈

## 🎯 주요 목표

### 고속병렬파일시스템 연동 기술 개발
- 고속병렬파일시스템 로부터 파일 생성/수정/삭제 이벤트 실시간 감지
- Webhook 기반 FileChangeEvent 처리
- 파일 속성 조회 및 초기 메타데이터 생성

### 멀티모달 메타데이터 자동 추출
- 텍스트/이미지/오디오/비디오 AI 기반 Feature 추출
- 공통·모달리티 확장 메타데이터 구조(JSONB) 설계

### 데이터 카탈로그 및 저장 구조 구축
- asset, dataset, embedding, asset_history 중심의 구조화된 스키마
- 변경 이력 관리 및 품질 관리 포함

### 고성능 검색 및 하이브리드 인덱싱
- Key-Value + Vector 기반 검색 통합
- pgvector + ivfflat 기반 벡터 인덱싱

## 🏗️ 시스템 아키텍처

아키텍처는 아래 세 레이어로 구성됩니다.

### 1️⃣ 데이터 연계 계층
- HPFS 파일 이벤트(Webhook) 수신
- 초기 메타데이터 수집
- MQ(metadata_init_event) 게시

### 2️⃣ 메타데이터 관리 계층
- 파일 기본정보 + AI 기반 추출 정보 저장
- 카탈로그 표준 스키마에 따라 정규화
- metadata_update_event 발행

### 3️⃣ 검색·서비스 계층
- Key-Value 인덱스(B-Tree/GIN/BRIN)
- 벡터 인덱스(pgvector)
- BM25 + Cosine Similarity 기반 하이브리드 검색
- API Gateway 기반 검색 서비스 외부 제공

## ✨ 주요 기능

### 1. 파일 인식 및 데이터 연계
- HPFS 이벤트를 Webhook 통해 수신
- 파일 속성 조회 후 내부 표준 구조로 변환

### 2. 메타데이터 수집 및 자동 추출
- 텍스트/이미지/오디오/비디오 유형 감지
- AI 기반 메타데이터 생성
- 오류 발생 시 재시도 및 상태 트래킹

### 3. 데이터 카탈로그 관리
- 자산(asset), 데이터셋(dataset) 관리
- 버전 및 변경 이력(asset_history) 관리

### 4. 인덱싱 및 검색
- 키-값(GIN/B-Tree) 및 벡터(pgvector) 인덱스 구성
- 하이브리드 검색: 키워드 + 유사도 기반 순위 결합

### 5. API / UI 서비스
- 검색 API, 메타데이터 조회 API 제공
- API Gateway 기반 인증/권한 제어

### 6. 운영 및 모니터링
- 로그 수집/분석
- 서비스 상태 모니터링
- 장애 감지 및 알림
